# Complete Project Command Reference

---

# 1️⃣ Environment & Dependency Management (uv)

---

## Install dependencies from `pyproject.toml`

```
uv sync
```

## Add new package

```
uv add <package_name>
```

## Remove package

```
uv remove <package_name>
```

## Run any Python command inside virtual environment

```
uv run <command>
```

---

# 2️⃣ Core Django Project Commands

---

## Create new Django project

```
uv run django-admin startproject <project_name>
```

## Create new Django app

```
uv run python manage.py startapp <app_name>
```

## Run development server

```
uv run python manage.py runserver
```

## Run development server on custom port

```
uv run python manage.py runserver 8001
```

## Show all available Django commands

```
uv run python manage.py help
```

---

# 3️⃣ Database & Migrations

---

## Create migrations

```
uv run python manage.py makemigrations
```

## Create migrations for specific app

```
uv run python manage.py makemigrations <app_name>
```

## Create empty migration (for data migrations)

```
uv run python manage.py makemigrations --empty <app_name>
```

## Apply migrations

```
uv run python manage.py migrate
```

## Run specific migration forward

```
uv run python manage.py migrate <app_name> <migration_number>
```

## Reverse a migration

```
uv run python manage.py migrate <app_name> <previous_migration_number>
```

## Show migration status

```
uv run python manage.py showmigrations
```

## Inspect SQL generated by migration

```
uv run python manage.py sqlmigrate <app_name> <migration_number>
```

## Flush database (⚠ DANGER: deletes all data)

```
uv run python manage.py flush
```

## Open database shell

```
uv run python manage.py dbshell
```

## Open Django shell

```
uv run python manage.py shell
```

## Shell with auto-imports (if django-extensions installed)

```
uv run python manage.py shell_plus
```

---

# 4️⃣ Direct SQL Execution

---

## Run SQL file directly

```
uv run python manage.py dbshell < schema.sql
```

## Inside dbshell

SQLite:

```
.read file.sql
```

MySQL:

```
source file.sql
```

---

# 5️⃣ Data Backup & Restore

---

## Export entire database

```
uv run python manage.py dumpdata > data.json
```

## Export specific app

```
uv run python manage.py dumpdata <app_name> > app_data.json
```

## Load data

```
uv run python manage.py loaddata data.json
```

---

# 6️⃣ Static Files & WhiteNoise

---

## Collect static files (Production)

```
uv run python manage.py collectstatic --noinput
```

## Clear old static files before collecting

```
uv run python manage.py collectstatic --noinput --clear
```

## Collect with verbose output

```
uv run python manage.py collectstatic --noinput --verbosity 2
```

WhiteNoise works automatically when:

```
STATICFILES_STORAGE = "whitenoise.storage.CompressedManifestStaticFilesStorage"
```

### Production Static Flow

```
tailwind build
        ↓
collectstatic
        ↓
WhiteNoise serves compressed + hashed files
```

# 7️⃣ Tailwind (django-tailwind)

---

## First-time setup

```
uv run python manage.py tailwind install
```

## Development mode (auto rebuild)

```
uv run python manage.py tailwind start
```

## Production build

```
uv run python manage.py tailwind build
```

⚠ Always run:

```
tailwind build
```

BEFORE:

```
collectstatic
```

# 🔟 ASGI Deployment (Uvicorn)

Use ASGI when:

- WebSockets
- Async views
- Real-time systems

---

## Start ASGI server (Production)

```
uv run uvicorn exam_management_system.asgi:application \
  --host 0.0.0.0 \
  --port $PORT
```

## With multiple workers

```
uv run uvicorn exam_management_system.asgi:application \
  --host 0.0.0.0 \
  --port $PORT \
  --workers 4
```

---

# 1️⃣1️⃣ WSGI Deployment (Gunicorn)

Use WSGI when:

- Traditional Django apps
- No WebSockets required

---

## Basic start

```
uv run gunicorn exam_management_system.wsgi:application \
  --bind 0.0.0.0:$PORT
```

## Production-ready configuration

```
uv run gunicorn exam_management_system.wsgi:application \
  --bind 0.0.0.0:$PORT \
  --workers 4 \
  --timeout 120 \
  --access-logfile -
```
